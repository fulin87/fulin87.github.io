一个合格的定时任务应该是怎样的
event Posted on: 2021-11-25 event_available Edited on: 2021-12-12 folder In: java visibility Views: 

定时任务(Job)是一个非常常见的功能，很多系统都有，但是实现健壮，稳定的定时任务并不是很多人认为的那样那么简单

一个优秀的定时任务应该满足以下这些要求

  • 分布式环境下的多节点调度
  • 执行完毕要有信息反馈
  • 处理好可能存在的浪打浪问题
  • 要有逻辑扩展点

分布式环境下的多节点调度

定时任务的基本要求是定时运行，但是100%准时只是存在于理想中罢了。将定时任务的调度和具体的业务逻辑分离，保证调度中心的高可用是一个最佳实践

业务逻辑应该有一个被调度的统一入口，在统一入口做一些公共逻辑的处理，然后分发到具体的任务中执行。也就是使用策略模式。

运行完毕一定要有信息反馈

定时任务很多都是批处理任务，异步执行是常态，如果没有信息反馈，将无法知道执行的结果和造成的影响，因此一定要有信息反馈，这里的信息反馈可以是邮件，短信，数据库数据等。可以根据定时任务的重要程度选择不同的反馈方式。

浪打浪问题要妥善解决

所谓浪打浪问题，是指上一轮任务还没有执行完毕，下一轮任务就已经开始了。针对这种情况，在业务逻辑端要做好控制，区分出哪些任务是允许浪打浪，哪些任务是不允许浪打浪的。针对不允许浪打浪的定时任务，要做好分布式环境下运行状态识别
